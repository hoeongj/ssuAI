## 이번 세션의 critical 발견 (다시 derive 하지 말 것)

### 1. Pyxis 인증은 HEADER, COOKIE 아님

- **올바른 인증**: `Pyxis-Auth-Token: <opaque ~32-char value>` request header
- **올바른 endpoint**: `https://oasis.ssu.ac.kr/pyxis-api/1/seat-rooms?smufMethodCode=PC&branchGroupId=1` (기타: `/branches`, `/favorite-seats`, `/seat-room-reservable-dates`)
- **잘못된 가정 (이전 spec)**: `Cookie: ssotoken=<value>` + `/pyxis-api/api/smuf/reading-rooms` — 이건 SPA shell 로딩용 cookie, API 인증과 무관
- **401 응답 형식**: HTTP 200 + body `{"success":false,"code":"error.authentication.needLogin","message":"Please log in to use this service."}` — HTTP status 아니라 body 검사로 판별
- **Companion headers** (브라우저가 보내는 것; Pyxis/WAF가 일부 요구할 가능성): `Accept: application/json, text/plain, */*`, `Accept-Language: ko`, `Referer: https://oasis.ssu.ac.kr/library-services/smuf/reading-rooms`, `User-Agent` (Chrome UA string)

### 2. ssutoday 패턴은 Task 14에만 적용 가능, Task 13에는 X

- ssutoday는 RN WebView로 SmartID SSO redirect를 `onLoadStart` 가로채기 (네이티브라 SOP 우회 가능)
- 웹 ssuAI는 SOP가 popup의 `document.cookie`/`location.href`/postMessage 다 차단 → spec §7 #1 spike NEGATIVE
- Task 14 (u-SAINT)는 `apiReturnUrl`을 ssuAI 백엔드로 설정하면 SmartID가 우리 origin으로 302 → query param이 same-origin으로 도착 → SOP 우회 불필요. ssutoday 백엔드의 2-phase auth (validation + portal Jsoup) verbatim 포팅 가능.
- Task 13 (library)는 ssutoday 스타일 SSO 없음, oasis 자체 로그인 → manual capture 필요

### 3. 2026 트렌드 정리 (포트폴리오 narrative 용)

- **MCP OAuth 2.1 Spec (2025-11-25)** — MCP 서버는 OAuth 2.1 Resource Server. RFC 8707 Resource Indicators 의무. *Pyxis는 OAuth 미지원 — 그래서 원칙만 따옴.*
- **Phantom Token Pattern** — Agent (LLM)는 raw credential 절대 노출 X. 짧은 per-session 토큰만 보고, 짧은 시점에 proxy가 진짜 credential로 swap.
- **Browser Session Delegation** (Manus / Claude Browser Extension / Chrome M144 --autoConnect / Bitwarden MCP) — 사용자 기존 브라우저 세션 위임.
- **ssuAI 적용 narrative**: "Modern MCP servers delegate via OAuth 2.1; SSU portals are legacy proprietary auth, so we adapt the *principle* of phantom-token (LLM never sees credentials, only connector layer holds token briefly) through manual capture that respects browser SOP." ADR 0013에 정리.
