## 다음 세션 액션 (순서대로)

### 1. Task 14 backend 마무리 — 2 PR 남음

**PR 14b-3 — `SaintSsoService` (ssutoday Phase 1/2 port)**

ssutoday 의 `packages/server/spring-backend/src/main/java/.../service/AuthServiceImpl.java`
의 `uSaintAuth` 메서드 그대로 포팅:

- 입력: `sToken`, `sIdno` (one-shot tokens from SmartID redirect)
- Phase 1: GET `https://saint.ssu.ac.kr/webSSO/sso.jsp?sToken=...&sIdno=...`
  with `Cookie: sToken=...; sIdno=...`. 응답 body 에 `location.href = "/irj/portal"` 포함 여부 검증. `Set-Cookie` 헤더 모두 수집 (portal session cookies).
- Phase 2: GET portal URL (configurable) with phase 1 cookies. Jsoup
  으로 학번/이름/소속/학적상태 파싱 — ssutoday 의 `main_box09` /
  `main_box09_con` 셀렉터 그대로.
- 출력: `UsaintAuthResult(studentId, name, major, enrollmentStatus)`
  record (sToken/sIdno 는 메서드 밖으로 안 나감, 메서드 종료 후 GC).
- `SaintSsoProperties` (@ConfigurationProperties): `saint-sso-url`,
  `saint-portal-url`, `timeout`. env var override.
- 예외: phase 1 실패 → `SaintAuthFailedException`. phase 2 파싱 실패 →
  `SaintPortalUnavailableException`. controller 가 302 query param 으로
  매핑.
- 테스트: ssutoday 의 실제 응답 fixture (PII 스크럽: 학번/이름 가짜로) +
  MockWebServer 로 phase 1/2 round-trip. ServiceTest 4~6 케이스.

`global/exception/` 에 위 두 예외 클래스 추가. 그러나 `GlobalExceptionHandler`
매핑은 안 함 — SsoCallbackController 가 query-param redirect 로 직접 처리.

**PR 14b-4 — `SaintSsoCallbackController` + auth endpoints + 통합**

- `GET /api/auth/saint/sso-init` — 302 to SmartID, query string 의
  `apiReturnUrl` 은 backend 가 구성 (`${SSUAI_API_BASE_URL}/api/auth/saint/sso-callback`).
  frontend 는 이 endpoint 만 호출. (사용자 spike 통과 가정 — §10 #1)
- `GET /api/auth/saint/sso-callback?sToken=...&sIdno=...` —
  `SaintSsoService.authenticate(sToken, sIdno)` → `StudentService.upsertOnLogin`
  → `JwtProvider.issueAccess/issueRefresh` → 302 to
  `${SSUAI_FRONTEND_ORIGIN}/auth/return?ok=1` + refresh cookie 설정
  (HttpOnly, Secure, SameSite=Lax, Path=/api/auth).
- `POST /api/auth/refresh` — refresh cookie 검증 → access 새로 발급.
- `GET /api/auth/me` — Authorization Bearer access JWT → 현재 Student.
- `JwtAuthFilter` (OncePerRequestFilter) — Authorization 헤더 파싱 →
  request attribute `ssuai.studentId` 박음. Spring Security 도입 안 함 —
  controller 가 attribute 읽음.
- ADR 0014 — redirect-callback 패턴 (SOP 우회 안 되는 web 에서 작동
  원리).
- 테스트: Controller @WebMvcTest 슬라이스 + sso-callback 통합
  (MockWebServer 로 saint 흉내).
- application-prod.yml: `SSUAI_API_BASE_URL`, `SSUAI_FRONTEND_ORIGIN`,
  `SSUAI_JWT_SECRET` env var 요구 명시.
