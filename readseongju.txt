발견된 이슈와 처리 결과:

[수정됨 ✅]
  1. LlmChatService.java — 시스템 프롬프트 오류
     "LMS 과제, 수강신청, 졸업요건 같은 시스템 연동은 아직 지원 안 함"
     → get_my_assignments 와 get_my_library_loans 는 이미 동작 중인데
       여전히 미지원이라고 써져 있었음.
     수정: "수강신청, 졸업요건 같은 시스템 연동은 아직 지원 안 함"으로 교체.
     인증 필요 도구 목록도 모두 정확하게 업데이트.

  2. LlmChatService.java — SECRET_GUIDANCE 범위 설명 오류
     "지금은 학식, 기숙사 식단, 캠퍼스 시설, 도서관 좌석/도서 검색만 도와줄 수 있어요"
     → 개인화 도구들(시간표, 성적, 과제, 대출)이 이미 지원되는데 빠져 있었음.
     수정: 실제 지원 범위로 업데이트.

  3. WeeklyMealSkeleton.tsx — React key anti-pattern
     <Skeleton key={index} ...> → <Skeleton key={`skeleton-day-${index}`} ...>
     index 를 key 로 쓰면 React가 DOM 재사용을 못 할 수 있어.
     semantic한 string key로 교체.

[검토 후 유지 🔍]
  4. LlmChatService.java — looksLikeSecretInput()에서 "토큰"/"token" 차단
     audit agent가 "라이브러리 토큰 입력을 막을 수 있다"고 지적했는데,
     실제 Pyxis-Auth-Token 입력은 LibraryLoginModal (전용 UI)을 통해서만
     이루어지도록 설계됨. chat에서 토큰 언급 차단은 의도된 보안 정책. 유지.

  5. RealSaintScheduleConnector.java — @SuppressWarnings("unused") Clock 필드
     "future cache TTL keying" 주석이 있고 패키지 전용 테스트 생성자에서
     Clock을 주입받음. 실제 활용 경로가 확보됨. 유지.

  6. useSaintAuth.tsx — console.warn() 2개
     401이 아닌 예기치 않은 auth 오류에서만 발동. 정상 동작 경로 아님.
     삭제하면 오히려 프로덕션 디버깅이 어려워짐. 유지.

  7. ThreadLocal 3개 중복 패턴 (SaintToolContext, LmsToolContext, LibraryToolContext)
     패턴은 동일하지만 메서드명이 각각 다름 (withStudentId vs withSessionKey).
     제네릭 추상화 시 호출부 전체 수정 필요 → 현재 위험 대비 이득 없음. 유지.
     (각 컨텍스트가 독립적으로 동작해야 하는 이유도 있음)
